本文记录学校工资从信息维护到生成支付的实际操作顺序，适合作为月度办理时的步骤核对。财政一体化系统菜单、经济分类和审核要求可能调整，办理时应以江苏本地区当前系统和单位审核口径为准。

## 一、工资信息维护与送审

1. 进入“工资发放 → 工资信息维护”。
2. 在工资信息维护环节处理本月发生变化的人员和工资项目。
3. 分别核对工资批次和数币批次，重点检查人数与实发合计。
4. 工资和数币分别送审。
5. 使用 02KEY 完成对应审核。

## 二、工资生成支付与额度匹配

进入“工资生成支付”，选择单位和“工资”批次，然后进行额度匹配。

如有额度未匹配，按实际工资项目核对经济分类。当前工作记录使用以下对应关系：

1. 农村教师补贴、边远补贴：`30399 其他对个人和家庭的补助`。
2. 基础性绩效：`30107 绩效工资`。
3. 其余工资项目：`30101 基本工资`。

修改完成后保存，再生成支付。经济分类如有新通知，应以当期要求为准，不要仅凭历史记录直接套用。

## 三、生成数币支付

选择单位和“数币”批次，完成额度匹配后生成支付。工资和数币是两个批次，应分别检查人员范围和金额，避免重复或遗漏。

## 四、完成送审和审核

确认工资、数币支付均已生成并送审，再通过 02KEY 完成审核。结束前再次核对工资实发、数币金额和系统生成支付合计。

## 工资发错后的退回顺序

1. 进入“工资发放 → 已办结”，在右上角点击撤销。
2. 进入“生成支付”，在右上角点击退回。
3. 使用 02KEY 进入“业务流程 → 审核 → 被退回”，点击退回。
4. 回到工资信息维护，重新调整工资并按上述流程再次核对。
