**老九的工资系统**是一款面向学校工资业务的 Windows 桌面软件，重点围绕五件事展开：工资报账、调整五险一金基数、核算绩效工资、乡镇补贴调整、人事信息管理。

工资工作最麻烦的地方，往往不是某一张表，而是“数据来源多、人员信息要一致、金额口径要统一、结果还要能导入一体化系统”。这款软件的目标，就是把这些分散步骤整理成一套稳定的本地工作台，让每月报账、年度调整和人员核对更清楚。

## 适合哪些场景

- 每月需要处理工资表、社保、个税、公积金等材料，并生成工资报账结果。
- 年初或基数调整时，需要根据工资、社保、公积金数据核算五险一金基数。
- 需要核算绩效工资差异，识别晋级、小档、退休、调出、新进等变化。
- 需要按乡镇工作年限和对照标准调整乡镇补贴。
- 需要维护人事基础信息，让工资、预算、补贴和报表数据能互相核对。

## 核心模块

### 工资报账

工资报账模块用于处理每月工资发放相关材料。软件可以结合工资表、社保未申报汇总、个税计算表和单位基础设置，整理出报账所需的阶段结果和最终结果。

常见输出包括：

- 工资报账预览和历史记录
- 保险导入文件
- 凭证导入文件
- 工资导入文件
- 补发工资文件

软件会校验当前月份、社保所属期、个税所属期、人员匹配和关键金额差异，尽量把问题拦在生成之前。

### 调整五险一金基数

五险一金基数调整模块用于处理年初或阶段性基数调整。它可以结合工资表、公积金账号表、社保明细、个税模板和社保基数模板，辅助生成调整结果。

这一部分适合处理：

- 社保缴费基数核算
- 公积金基数调整
- 个税和社保相关模板整理
- 工资导入、住房申报、社保基数导入等文件生成
- 调整结果与本地工资数据的核对

### 核算绩效工资

绩效工资模块用于对比两个期间的工资数据，核算人员变化和工资项目变化。软件以证件号码等稳定字段进行匹配，减少同名、重名造成的误判。

它可以辅助识别：

- 正常在册人员的工资变化
- 晋级、小档、岗位或薪级变化
- 新进、调入、调出、退休等人员变化
- 两个期间工资数据缺失或异常项目

核算完成后，可以导出绩效工资差异结果，方便后续复核和留档。

### 乡镇补贴调整

乡镇补贴模块用于按工作年限、人员信息和补贴标准核算调整结果。对于需要补全身份证号、核对任职年限、生成新标准补贴表的场景，可以减少手工查找和重复计算。

适合处理：

- 乡镇工作年限对照
- 乡镇补贴标准调整
- 人员身份证号补全和匹配
- 调整结果导出和复核

### 人事信息管理

人事信息管理模块用于维护人员基础资料，并和工资、预算、补贴等数据互相校验。工资业务很多问题最终都会回到人员信息是否准确：姓名、证件号、岗位、学历、任教信息、职级简历等字段如果不统一，后面的报账、补贴和统计都会受到影响。

软件可以把人事信息作为工资业务的基础底座，用于支持：

- 人员基础信息维护
- 工资数据和人事信息一致性核对
- 乡镇补贴、绩效工资、统计报表等模块的数据匹配
- 差异确认后的数据修正和留档

## 使用思路

第一次使用时，建议先完成单位设置和人事信息整理，再导入基础工资数据。之后可以按业务场景进入对应模块：

- 每月发工资时，进入工资报账模块。
- 年初或基数变化时，进入五险一金基数调整模块。
- 需要核算差异时，进入绩效工资模块。
- 补贴标准变化时，进入乡镇补贴调整模块。
- 数据对不上时，回到人事信息管理模块核对人员底账。

这样每个模块不是孤立的小工具，而是围绕同一套人员和工资数据协同工作。
